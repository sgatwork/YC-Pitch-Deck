# Contributing to pitch-deck-skill

Thanks for wanting to improve this. Contributions fall into three categories:

---

## 1. Content improvements (`references/content.md`)

Good PRs here look like:
- Better slide format guidance based on what's actually working in YC interviews
- Improved examples of strong vs weak headlines, credentials, or market math
- New investor questions / frameworks worth folding in
- Corrections to anything that's factually off

**How to submit:** Edit the file, show the before/after in your PR description, and explain why the new version is better.

---

## 2. Design system improvements (`references/design.md`)

Good PRs here look like:
- Layout patterns for slide types not currently covered
- Better PptxGenJS code patterns (more reliable, cleaner output)
- QA checklist additions — things that slip through and cause visual bugs
- Alternative colour palettes with real examples showing they work

**How to submit:** If you're changing the visual system, include before/after slide screenshots in the PR.

---

## 3. Example decks (`examples/`)

If you've used this skill to build a real deck (anonymised is fine), adding it to examples helps others see what's possible.

**What makes a good example:**
- All 7 slides shown
- A non-generic company (something specific enough that the content decisions are instructive)
- Screenshot quality — 150 DPI minimum

---

## What we're not looking for

- Generic "AI pitch deck" templates with placeholder content
- Colour palette changes that don't come with a clear rationale
- Adding more text to slides — the whole point of this system is radical minimalism
- Framework additions that conflict with the one-idea-per-slide rule

---

## Process

1. Fork the repo
2. Make your changes in a branch
3. Open a PR with a clear description of what changed and why
4. A maintainer will review within a few days

No CLA, no formal process — just keep it simple and useful.
